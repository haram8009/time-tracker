# BlockMerge: 인접 동일 카테고리 블록 자동 병합

새 TimeBlock을 저장할 때, 앞·뒤에 같은 Category의 TimeBlock이 인접해 있으면 새 블록을 생성하지 않고 기존 블록을 확장·병합한다. 양쪽 모두 인접한 경우 세 블록을 하나로 합친다.

## Consequences

- **insert 대신 update:** `TimeBlockStore.insert` 호출 전 인접 블록 조회 → 병합 여부 결정 → `update` or `insert`.
- **덮어쓰기와의 관계:** 다른 카테고리 블록을 덮어쓰는 경우는 별개 동작(병합 없이 기존 블록 분할/삭제). 병합은 같은 카테고리일 때만 적용.
- **삭제 시 분리 불필요:** 병합된 블록을 편집/삭제할 때 사용자가 명시적으로 범위를 지정하므로 자동 분리 로직은 불필요.
